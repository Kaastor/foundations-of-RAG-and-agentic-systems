## Chapter concepts covered

- **Staged vs live knowledge snapshots** (implemented in code)

## What is implemented directly vs documented only

- **Replayable ingestion from source roots** - partially demonstrated. The system rebuilds from source roots; it does not stream source events.
- **Version-aware ranking / stale evidence handling** - partially demonstrated. Superseded documents are penalized and active+superseded conflicts are surfaced.

## Debugging tips

- Compare live results from a base-only snapshot to results after publishing the update snapshot.
- Inspect `workspace/live.json` and `workspace/snapshots/*/manifest.json` for versioned publishing state.

## Trace and log outputs to inspect

## Limitations / simplifications

Incremental updates are simulated by rebuilding from base + update roots. The repository does not implement streaming ingestion or blue-green infrastructure.
